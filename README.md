# AI Inbox

一个本地优先的 AI Inbox。

它不是传统的“先建项目、再建列表、再手动整理”的 Todo 工具，而是一个更接近日常输入习惯的收件箱：

- 你只管往一个输入框里写
- 原始内容先落本地
- AI 再在后台把内容整理成 `计划` 或 `笔记`
- 相近内容会自动归并到同一主题
- 需要提醒的计划可以补充或编辑提醒时间

这个项目适合那些“想快速记下来，整理交给系统”的工作流。

## 项目定位

日常工作里，很多内容并不是标准 Todo。

有的是待完成的任务，有的是一段问题分析，有的是阶段性总结，还有一些只是临时想到的零散想法。传统 Todo 工具通常要求用户先决定内容应该放在哪，再自己维护结构；而这个项目的核心目标，是把“输入”与“整理”拆开：

1. 用户先输入
2. 系统先保存原文
3. AI 再异步分类、归并、润色、更新摘要

这样能尽量减少整理成本，同时保留原始上下文。

## 核心能力

### 1. 一个输入框承接所有输入

首页只有一个统一输入框，用户不需要在输入前先切换到不同模块。  
如果用户已经明确知道这条内容是 `计划` 还是 `笔记`，也可以手动指定类型，降低 AI 分类误差。

### 2. 本地优先

所有输入会先保存到本地数据库，再进入 AI 整理队列。

- Web 端存储使用 IndexedDB（Dexie）
- 桌面端通过 Tauri 封装
- 即使 AI 配置缺失或请求失败，原始内容也不会丢

### 3. AI 负责分类与归并

AI 的职责不是“替你写长文”，而是做收件箱整理：

- 判断这条内容更像 `计划` 还是 `笔记`
- 判断它是否应该归入已有主题
- 为内容生成更适合展示的标题
- 更新主题摘要

其中：

- `计划` 更强调可执行、可完成、可追踪
- `笔记` 更强调记录、分析、总结、灵感沉淀

### 4. 计划与笔记分开呈现

这个项目最终把高频工作内容收敛为两类：

- `计划`：强调推进、状态、完成、提醒
- `笔记`：强调沉淀、整合、润色、总结

相比把栏位拆得很细，这种结构更适合 AI 做稳定判断，也更符合真实工作场景。

### 5. 计划支持完成状态与提醒

对于计划类内容，支持：

- 标记完成 / 恢复未完成
- 已完成内容自动收敛到后面
- 根据文本识别提醒意图
- 手动设置、修改、关闭提醒时间
- 桌面通知提醒

### 6. 桌面客户端体验

项目基于 Tauri 2 提供桌面端能力：

- 启动后可隐藏到系统托盘
- 支持桌面通知
- AI 请求走桌面侧能力，避免浏览器跨域限制
- 支持在运行时修改模型配置

## 适合什么人

- 想要一个“先记下来，之后再整理”的个人工作台
- 任务、想法、分析、总结经常混在一起
- 不希望把数据默认放到云端
- 希望替换自己的 OpenAI-compatible 模型接口

## 快速开始

### 1. 安装依赖

```bash
pnpm install
```

### 2. 启动前端开发环境

```bash
pnpm dev
```

### 3. 启动桌面开发环境

```bash
pnpm tauri:dev
```

### 4. 打包桌面客户端

```bash
pnpm tauri:build
```

## AI 配置

复制 `.env.example` 为 `.env.local`，再填写你自己的接口配置。

示例：

```bash
VITE_AI_BASE_URL=/api/ai
VITE_AI_MODEL=your-model-name
VITE_AI_API_KEY=
AI_PROXY_TARGET=https://your-openai-compatible-endpoint.example/v1
AI_PROXY_API_KEY=
```

说明：

- 浏览器开发模式下，可以使用 `/api/ai` 代理以避免 CORS 问题
- 桌面客户端建议直接使用完整 `Base URL + API Key`
- 如果没有配置 AI，应用仍然可以用本地兜底规则工作

## 桌面端说明

当前项目使用 Tauri 2。

桌面端已包含：

- 系统托盘
- 本地通知
- 启动隐藏
- AI 运行时配置

如果你要构建不同平台，请在对应平台上准备相应工具链：

- Windows：Rust + Visual Studio Build Tools
- macOS：Rust + Xcode

## 数据与隐私

这个项目的设计原则是“原始输入先保留，本地数据优先”。

- 输入内容先写入本地
- AI 整理失败时仍保留原文
- 原文默认不作为主展示内容，但可以展开查看
- 主题摘要与展示标题属于 AI 整理结果，原始内容不会被覆盖

## 当前版本的设计取舍

为了让 AI 判断更稳定，当前版本刻意收敛为两种主要结果：

- 计划
- 笔记

这意味着它不是一个功能无限扩展的“全能知识系统”，而是优先保证：

- 输入轻
- 整理快
- 结构少
- 主题清晰
